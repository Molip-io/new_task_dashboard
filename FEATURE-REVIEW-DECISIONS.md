# 업무현황 대시보드 기능 적합성 검토 반영서

기준일: 2026-07-15  
검토 결론: 부분 적합 · P0 반영 후 v1 진행

## 1. 최종 제품 목적

이 대시보드는 업무를 많이 관리하거나 개인을 평가하는 도구가 아니다. 대표가 프로젝트의 현재 위치와 계획 대비 상태를 이해하고, 의사결정 지연과 팀 대기를 가장 먼저 발견하도록 돕는 읽기 전용 도구다.

첫 화면에서 다음 질문에 답할 수 있어야 한다.

1. 오늘 무엇을 결정해야 하는가?
2. 지금 실제로 무엇이 막혀 있는가?
3. 어제와 비교해 무엇이 달라졌는가?
4. 이 판단은 어느 범위까지 믿을 수 있는가?

## 2. v1 P0 확정

| P0 | 확정 내용 | 데이터가 없을 때의 처리 |
|---|---|---|
| 계획 대비 기준선 | 핵심 스펙별 범위 확정 예정일·제작 완료 예정일 | 정상·지연을 판정하지 않고 `기준선 미등록` |
| 의존관계 커버리지 | 확정 관계 또는 관계 없음 확인을 핵심 스펙별로 집계 | `병목 없음` 대신 `판단 정보 부족` |
| 데이터 신선도 | Notion·Slack·회의록별 마지막 성공 시각과 부분 실패 | 실패 출처와 마지막 성공값을 함께 표시 |
| 전일 스냅샷 | 상태·기한·담당·완료율·의존관계·결정 상태를 날짜별 저장 | 변화 목록을 만들지 않고 비교 불가 이유 표시 |

## 3. 기존 기능 보완 결정

### 대표 브리핑

- `오늘 결정할 것`에는 대표의 선택이 후속 작업을 바꾸는 사업 안건만 승격한다.
- 사용자 매핑, 수집 실패, 관계 미입력은 `확인 필요`에만 둔다.
- 최대 5건만 표시하며 질문·기한·영향 스펙·후속 작업·근거를 보여준다.
- 화면에서 결정을 선택하거나 저장하지 않는다.

### 건강상태

- 완료율을 건강상태 계산에 사용하지 않는다.
- 핵심 스펙 미확정, 계획 기준선 초과, 확정 의존관계, 실제 대기, 후속 작업 영향으로 규칙 엔진이 판정한다.
- 모든 프로젝트에 같은 `D-14` 같은 절대 기준을 적용하지 않는다. 각 스펙의 기준선과 확인된 후속 작업 소요기간을 비교한다.
- 기준선 또는 관계 데이터가 부족하면 `정상`이 아니라 `판단 제한`을 별도 표시한다.

### 담당별 흐름

- 기본 정렬은 `내 작업을 기다리는 확인된 후속 작업 수`다.
- 관계가 미측정된 사람은 하나의 그룹으로 묶고, 그 안에서만 열린 자식 일감 수를 참고 순서로 사용한다.
- 일감 수는 작게 표시하고 색상·순위로 과부하나 성과를 암시하지 않는다.
- 미배정 일감은 개인에게 귀속하지 않고 팀 큐에 통합한다.

### 확인 필요

- 상태·기한·담당·범위·의존관계·결정·블로커 등 핵심 필드 충돌만 생성한다.
- 출처별 주장과 원문을 함께 표시하고 시스템이 최신값을 정답으로 선택하지 않는다.
- 동일 항목이 대표 판단도 필요하면 브리핑에는 질문과 영향만, 확인 필요에는 출처 비교만 보여준다.
- Slack 채널 활동량 차이처럼 판단에 직접 필요하지 않은 정보는 확인 큐가 아니라 로그로 남긴다.

### GPT 요약

- 첫 화면 GPT는 여러 프로젝트 사이의 결정·병목·변화 우선순위를 압축한다.
- 프로젝트 GPT는 해당 프로젝트의 목표·현재 단계·계획 대비·후속 영향을 설명한다.
- 두 위치에서 같은 문장을 반복하지 않는다.
- 모든 문장에 근거 ID를 연결하고 실패 시 규칙 facts의 고정 템플릿으로 대체한다.

## 4. 첫 화면 확정 순서

0. 수집 신뢰 상태: 출처별 성공 시각, 부분 실패, 의존관계 커버리지, 전일 스냅샷
1. 오늘 결정할 것: 질문, 기한, 영향, 근거 · 최대 5건
2. 현재 막힌 것: 실제 대기, 재작업 위험, 검수 대기 후보, 판단 정보 부족
3. 어제와 달라진 것: 전일 스냅샷과 동일 필드 비교 결과

프로젝트·스펙, 담당별 흐름, 확인 필요는 첫 판단 이후 상세 탐색 화면으로 둔다.

## 5. 검토 의견 중 조정한 사항

1. GPT를 프로젝트 상세에만 두지 않는다. 대표의 30초 이해를 위해 첫 화면에도 두되, 규칙 facts를 설명하는 역할로 제한한다.
2. `D-14 미확정` 같은 고정 임계값을 사용하지 않는다. 프로젝트별 기준선과 후속 작업 소요기간을 사용한다.
3. 수집 실패 Slack 알림은 화면 기능이 아니라 운영 규칙으로 분리한다. 화면은 실패 상태와 마지막 성공 시각만 책임진다.
4. 미배정 큐, 병목 영향도, GPT 근거·대체 출력, 프로젝트 목표는 별도 메뉴로 만들지 않고 기존 화면에 통합한다.

## 6. v1 완료 조건

- P0 네 항목이 실제 데이터 계약과 화면에 연결된다.
- 기준선·의존관계·스냅샷이 없을 때 거짓 정상·거짓 병목 없음·가짜 변화를 만들지 않는다.
- 첫 화면의 세 영역이 결정 → 막힘 → 변화 순서로 보인다.
- 담당별 화면은 모든 개인과 자식 일감을 보여주되 팀 대기 영향이 일감 수보다 먼저 해석된다.
- 확인 필요와 오늘 결정할 것의 중복 목적이 제거된다.
- GPT 실패 시에도 동일한 규칙 사실을 읽을 수 있다.
